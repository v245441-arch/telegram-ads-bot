# Инструкция по запуску тестов для Telegram-бота

## Описание

Файл `test_bot.py` содержит комплексный набор тестов для проверки всех основных функций Telegram-бота объявлений. Тесты имитируют действия пользователя и проверяют корректность работы бота без использования реального Telegram API.

## Требования

Для запуска тестов необходимо установить следующие зависимости:

```bash
pip install -r requirements.txt
```

Основные зависимости:
- `aiogram` - для работы с Telegram Bot API
- `sqlite3` - встроенная библиотека Python для работы с SQLite
- `unittest` - встроенная библиотека Python для тестирования
- `unittest.mock` - для создания моков и заглушек

## Запуск тестов

### Способ 1: Запуск через Python напрямую

```bash
python test_bot.py
```

### Способ 2: Запуск через unittest

```bash
python -m unittest test_bot.py -v
```

### Способ 3: Запуск через pytest (если установлен)

```bash
pytest test_bot.py -v
```

## Структура тестов

Тесты разделены на 12 основных категорий:

1. **Добавление объявления** - полный цикл добавления с проверкой всех полей
2. **Просмотр списка объявлений** - проверка отображения всех полей и форматирования
3. **Просмотр по категориям** - фильтрация объявлений по категориям
4. **Поиск** - поиск объявлений по ключевым словам
5. **Личный кабинет** - просмотр объявлений пользователя
6. **Редактирование** - изменение всех полей объявления
7. **Удаление** - удаление объявления с подтверждением
8. **Избранное** - добавление/удаление объявлений из избранного
9. **Жалобы** - отправка жалоб и уведомление администратора
10. **Статистика** - проверка доступа к статистике для администратора
11. **Поддержка** - отправка сообщений в поддержку
12. **Проверка прав доступа** - проверка ограничений доступа

## Особенности тестирования

### Временная база данных

Тесты используют временную базу данных (`test_ads.db`), которая:
- Создается автоматически перед каждым тестом
- Очищается после каждого теста
- Удаляется после завершения всех тестов
- Не затрагивает основную базу данных

### Моки и заглушки

Для тестирования функций, требующих взаимодействия с внешними сервисами:
- Используются моки для имитации Telegram Bot API
- Заглушки для проверки отправки сообщений администратору
- Имитация callback-запросов

### Проверка данных

Каждый тест проверяет:
- Корректность сохранения данных в базе данных
- Правильность форматирования выводимого текста
- Соответствие ожидаемым результатам
- Обработку ошибок и исключений

## Интерпретация результатов

- **Пройдено** - тест выполнился успешно
- **Провалено** - тест завершился с AssertionError (логическая ошибка)
- **Ошибки** - тест завершился с исключением (техническая ошибка)
- **Пропущено** - тест был пропущен (если используется)

## Возможные проблемы

### Проблема: Импорт модуля bot.py

Если возникают ошибки импорта:
1. Убедитесь, что файл `bot.py` находится в той же директории, что и `test_bot.py`
2. Проверьте, что в `bot.py` нет синтаксических ошибок
3. Убедитесь, что все зависимости установлены

### Проблема: Ошибки базы данных

Если возникают ошибки SQLite:
1. Проверьте права на запись в директорию
2. Убедитесь, что нет блокировок файлов
3. Попробуйте запустить тесты с правами администратора

### Проблема: Ошибки импорта aiogram

Если возникают ошибки aiogram:
1. Убедитесь, что aiogram установлен: `pip install aiogram`
2. Проверьте версию aiogram (требуется 3.x)
3. Убедитесь, что нет конфликтов с другими версиями
